# Deployment Checklist

Use this checklist to ensure your Task Manager API is production-ready before deployment.

## Pre-Deployment

### Code & Configuration
- [ ] All changes committed to git
- [ ] `.env` file NOT committed (in .gitignore)
- [ ] `.env.example` has no secrets
- [ ] `DEBUG=False` in production
- [ ] `SECRET_KEY` is unique per environment
- [ ] `ALLOWED_HOSTS` configured correctly
- [ ] `CSRF_TRUSTED_ORIGINS` configured
- [ ] No hardcoded API keys or passwords
- [ ] No print() debugging statements

### Security
- [ ] Run `python manage.py check --deploy`
- [ ] HTTPS configured
- [ ] Security headers set in settings.py
- [ ] Password validation enabled
- [ ] CSRF protection enabled
- [ ] SQL injection prevention verified
- [ ] XSS protection enabled
- [ ] Database password is strong

### Database
- [ ] Migrations up to date: `python manage.py migrate`
- [ ] Database backup strategy planned
- [ ] PostgreSQL configured (not SQLite)
- [ ] Connection string correct
- [ ] Database credentials secure
- [ ] SSL connection enabled for database

### Dependencies
- [ ] `requirements.txt` up to date
- [ ] All imports working
- [ ] No unused dependencies
- [ ] All security patches applied
- [ ] Tested with `pip install -r requirements.txt`

### Testing
- [ ] Unit tests pass: `python manage.py test`
- [ ] No test failures
- [ ] Code coverage acceptable (>80%)
- [ ] Edge cases tested
- [ ] Error handling tested
- [ ] API endpoints tested

### Static Files
- [ ] Static files collected: `python manage.py collectstatic --noinput`
- [ ] WhiteNoise configured
- [ ] CSS and JS loading correctly
- [ ] Admin interface looks good
- [ ] Swagger/ReDoc documentation loads

### Documentation
- [ ] README.md is accurate
- [ ] API documentation complete
- [ ] Deployment guide followed
- [ ] Contributing guide complete
- [ ] Security guide reviewed
- [ ] Architecture documented

## Render Deployment Specific

### Configuration
- [ ] `render.yaml` configured correctly
- [ ] Environment variables set in Render dashboard
- [ ] `SECRET_KEY` generated by Render
- [ ] PostgreSQL add-on attached (if using)
- [ ] Database migrations set to run automatically

### Verification
- [ ] Blueprint applied successfully
- [ ] Deployment logs show no errors
- [ ] Service URL accessible
- [ ] Swagger UI loads at `/swagger/`
- [ ] API endpoints responding
- [ ] Authentication working
- [ ] Database queries working

### Post-Deployment
- [ ] Check application logs for errors
- [ ] Verify HTTPS redirect working
- [ ] Test API endpoints with authentication
- [ ] Create superuser (if needed)
- [ ] Verify static files serving
- [ ] Monitor resource usage

## Docker Deployment Specific

### Build & Test
- [ ] Dockerfile builds successfully: `docker build -t task-manager-api .`
- [ ] Docker image runs: `docker run -p 8000:8000 task-manager-api`
- [ ] docker-compose up works: `docker-compose up -d`
- [ ] Services start without errors
- [ ] Logs show no critical errors

### Configuration
- [ ] `.dockerignore` configured
- [ ] Environment variables in `docker-compose.yml`
- [ ] Volumes mounted correctly
- [ ] Network configuration correct
- [ ] Health checks passing

### Production
- [ ] Production docker-compose ready
- [ ] PostgreSQL persistence configured
- [ ] Static files volume mounted
- [ ] Nginx reverse proxy configured
- [ ] SSL certificates available

### Ongoing
- [ ] Weekly security updates check

- [ ] Quarterly security audit
- [ ] Backup verification

## Rollback Plan

In case of issues:

1. **Render**
   - [ ] Previous deployment available
   - [ ] Can revert to previous version
   - [ ] Database backups available

2. **Docker**
   - [ ] Previous image tagged
   - [ ] Can rollback with `docker-compose down && docker-compose up`
   - [ ] Database backup procedure known

3. **Communication**
   - [ ] User notification plan ready
   - [ ] Team notified of rollback procedure
   - [ ] Documentation of issues kept

## Success Criteria

✅ Application loads without errors
✅ All API endpoints responding

✅ Database queries successful
✅ Static files serving correctly
✅ HTTPS working
✅ Response times acceptable
✅ No critical errors in logs
